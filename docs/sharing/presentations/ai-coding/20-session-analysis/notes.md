最后一页，我想用数据来验证一下——刚才说的这些实践，到底长什么样？

我们分析了 57 个 session、834 条用户消息。结果很有意思。

占比最高的操作是什么？不是"做这个"、"改那个"。是 **"做完了吗？确定吗？"** ——占了 32%。第二高的是"跑测试"——23%。第三是"组织 review"——15%。

换句话说，**在 AI Coding 的模式下，你的角色从"写代码的人"变成了"验证代码的人"。**

大家看下面这个对比。典型用户是一句话描述需求，信任输出，修完即走。而我们的方式是：粘贴完整 Spec，每步验证，举一反三补测试，最后还要 commit、push、release、更新文档。
