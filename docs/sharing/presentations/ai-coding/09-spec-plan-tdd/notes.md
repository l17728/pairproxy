这三个我合并讲，因为它们本质上是一条流水线：先想清楚要做什么，再想清楚怎么做，最后严格按步骤来。

Spec 里面有一个要素特别容易被忽略——"已拒绝方案"。AI 特别喜欢走捷径。如果你不告诉它"不要用方案 X，因为 Y"，它大概率会选 X。

然后是 TDD。TDD 的关键不是"先写测试"——关键是**先跑一遍测试，确认它确实失败了**。

为什么？因为 AI 写的测试，有可能永远不会失败。比如它把不该 mock 的东西 mock 了。如果你不验证失败，这个测试就是假的安全感。

我的原则很简单：**永远不要让 AI 直接开始写代码。** Spec → Plan → TDD，这个顺序不能跳。
