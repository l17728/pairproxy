这些都是 Git 历史里留下的真实痕迹。

有一个我觉得特别有意思：AI 对 AI 的批评。我让 AI 写了一份 335 行的 Review 文档，给初稿打了 55 分，然后又让另一个 AI 对这份 Review 本身打分——35 分。

这说明什么？**多层 AI 审核是真的有价值的。** 不是因为 AI 比人强，而是因为不同的 AI 在不同的上下文窗口里，看到的东西不一样。

还有一次，2 小时内改了 8 个模块——db、alert、api、proxy、cli、config——因为我让多个 agent 并行跑。

所以善用 Git 记录。你的 commit message、review 文档、验收报告，不只是过程证据，更是你复盘的原材料。
