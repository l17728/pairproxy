快速过一下版本演化。从 v1.0 的基础代理，到 v2.24 的企业级平台。

注意这个路径：v1.x 先把核心跑通——HTTP 代理、SSE 流式、Token 统计。v2.0 开始加企业特性——认证、配额、Dashboard。v2.9 做协议转换，这是最复杂的部分。v2.11 之后是集群、PostgreSQL、安全性。

为什么这么排？因为 **AI Coding 的项目也要增量演进**。不要让 AI 一次做太多。每个版本聚焦一个主题，做完验收再开下一个。这样出错的时候你能快速定位——"这个 bug 是 v2.9 引入的，因为那个版本在做协议转换。" 如果所有功能都是 v2.0 一起堆上去的，你根本无从查起。
