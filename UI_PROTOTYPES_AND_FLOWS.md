### 批量解决告警流程

```
1. 勾选多行或点 [☑️ 全选] ↓

2. 底部 [🗑️ 批量解决 (N 条)] 按钮激活 ↓

3. 点击 [🗑️ 批量解决 (4 条)] ↓

┌──────────────────────────────────────────────┐
│ ⚠️  确认                                      │
├──────────────────────────────────────────────┤
│                                              │
│ 您将解决以下 4 条告警:                        │
│  • api.openai.com (error, Critical)          │
│  • api.anthropic.com (warning, Warning)      │
│  • api.local.com (degraded, Error)           │
│  • api.test.com (error, Critical)            │
│                                              │
│ [确认解决]  [取消]                           │
│                                              │
└──────────────────────────────────────────────┘

点 [确认解决] ↓

4. POST /dashboard/alerts/resolve-batch
   body: ids[]=alert1&ids[]=alert2&ids[]=alert3&ids[]=alert4

5. 302 redirect /dashboard/alerts?tab=active&flash=已解决4条告警

6. 页面刷新:
   - 表格清除已解决告警行
   - 顶部统计数字更新 (共 3 条 → 共 3 条)
   - checkbox 重置
   - 成功提示消息消失 (5s 后自动)
```

## 完整交互流程

### 场景 1：新建 Group-Target Set 并绑定分组

```
🎬 场景: 工程师为 engineering 分组新建专属目标集（优先使用国内服务）

【Step 1】进入 LLM 管理页面
  用户点导航栏 [LLM]
  → GET /dashboard/llm?tab=targetsets (默认进入目标集 Tab)
  ← handler 返回 llm.html 带 ActiveTab=targetsets

【Step 2】点击 [+ 新建目标集]
  右栏展开模态框

【Step 3】填写表单
  名称: engineering-targets (CN)
  绑定分组: engineering [选择 ▼]
  策略: 轮询 (round_robin)
  重试: 顺延 (try_next)

  初始成员:
    行1: api.local.cn        权重:5  优先级:0
    行2: api.openai.com     权重:2  优先级:1

  点 [创建目标集]

【Step 4】表单提交
  POST /dashboard/llm/targetsets
  body:
    name: engineering-targets (CN)
    group_id: eng-group-id
    strategy: round_robin
    retry_policy: try_next
    members: [
      { target_url: api.local.cn, weight: 5, priority: 0 },
      { target_url: api.openai.com, weight: 2, priority: 1 }
    ]

【Step 5】后端处理
  ✓ 1. groupTargetSetRepo.Create() → 新建 GroupTargetSet
  ✓ 2. groupTargetSetRepo.AddMember(setID, member1)
  ✓ 3. groupTargetSetRepo.AddMember(setID, member2)
  ✓ 4. (可选) 调用 SemanticRouter.UpdateTargetSetMapping() 更新路由

【Step 6】页面跳转
  302 /dashboard/llm?tab=targetsets&selected={newSetID}&flash=目标集已创建

【Step 7】用户看到
  ✅ 模态框关闭
  ✅ 左栏列表显示新的 engineering-targets (CN)
```

## 前端状态管理约定

由于采用 SSR + 原生 JS，无状态管理库。约定如下：

### URL 状态

| 参数 | 值 | 作用 |
|------|-----|------|
| `tab` | `targetsets` \| `live` \| `active` | 标记当前 Tab |
| `selected` | `{setID}` | 标记选中的目标集（右栏显示） |
| `page` | `1,2,3...` | 分页（历史查询 Tab） |
| `days` | `7,30,90` | 时间范围（历史查询 Tab） |
| `flash` | `{message}` | 成功/失败提示消息（自动消失） |

示例：
```
/dashboard/llm?tab=targetsets&selected=eng-set-001&flash=成员已更新
/dashboard/alerts?tab=history&days=30&page=2
```

### DOM 状态

| 元素 | 状态变化 | 触发方式 |
|-----|---------|---------|
| 成员行编辑按钮 | disabled → enabled | 列表行编辑模式切换 |
| 批量解决按钮 | disabled → enabled | checkbox 计数 > 0 |
| 全选 checkbox | 不定→checked | 点击全选按钮 |

所有 DOM 状态通过 CSS class 切换（`.edit-mode`, `.disabled`, `.active` 等）。

### 本地存储

**不使用** localStorage 或 sessionStorage。理由：
- Tab 状态存 URL，浏览器原生支持后退
- 用户操作直接 302 redirect，无需保存临时态
