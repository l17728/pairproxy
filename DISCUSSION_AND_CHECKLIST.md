#### Q1: 是否同意采用 "Tab 内嵌" 方案扩展 LLM 和告警页面？

**方案简述**：
```
当前: /dashboard/llm → 单一页面
目标: /dashboard/llm?tab=targets|targetsets|bindings → 三标签页共存

当前: /dashboard/alerts → 实时流单视图
目标: /dashboard/alerts?tab=live|active|history → 三标签页共存
```

**优点**：
- 导航栏保持现有 8 项简洁设计（无膨胀）
- 深链接可分享，浏览器后退可用
- SSR 框架支持，无需改构建

**缺点**：
- 单页面逻辑复杂度上升
- Tab 切换无动画感（纯服务端渲染）

**替代方案**（如不同意）：
- 新增独立页面 `/dashboard/targetsets` 和 `/dashboard/alerts/history`
- 导航栏需 dropdown menu 或分为两行

**建议**：✅ 采用 Tab 方案（简洁性 > 动画效果）

#### Q2: Group-Target Set 管理页面的信息架构

**两个设计选项**：

**选项 A（推荐）：双栏式 + 右侧详情**

```
左栏(1/3):           右栏(2/3):
目标集列表卡片  ←→  选中集的详情 + 成员管理
- 列表项卡片
- 每项显示:
  * 名称
  * 成员数
  * 策略
  * 绑定分组
  * [编辑][删除]
```

**优点**：
- 多个目标集快速切换
- 选中项视觉反馈清晰
- 成员管理集中在右栏

**选项 B（替代）：列表视图 + 行内编辑**

```
单栏列表:
目标集名称  成员数  策略  绑定分组  [编辑][删除][详情]
  ↓
点 [详情] 弹出模态框
```

**优点**：
- 页面更简洁
- 响应式更好（移动设备）

**缺点**：
- 无法同时看多个集的信息
- 成员编辑需进模态框

**建议**：✅ 选项 A（最符合现有 UI 设计风格）

**您的决策**:
- [ ] 选项 A（双栏）
- [ ] 选项 B（列表）

#### Q3: 成员健康状态的实时性

**问题**：`GroupTargetSetMember.health_status` 由后台 TargetHealthMonitor 30s 更新一次。前端显示会有延迟。

**三个解决方案**：

**方案 1（轻量）：页面初始加载后不实时更新**
```
- 用户访问 /dashboard/llm?tab=targetsets
- 页面加载时，展示成员健康状态（数据来自 DB，可能 ≤30s 旧）
- 用户手动刷新页面获取最新状态
```
**成本**: 最低（无新代码）
**适用**: 中低频次访问

---

**方案 2（中等）：添加"刷新"按钮**
```
右栏 [刷新] 按钮 → 重新加载当前选中目标集的成员列表
fetch /api/admin/targetsets/{id}/members
```
**成本**: 2-3 小时（加 API 端点 + 前端 JS）
**体验**: 用户可主动刷新，获得最新数据

---

**方案 3（完整）：SSE 实时推送**
```
- 当后台 health_status 更新时，广播 SSE 事件
- 前端监听，自动更新表格行
```
**成本**: 1-2 天（需要改后台告警系统架构）
**体验**: 实时同步，最好

---

**建议**：✅ 方案 1 or 2（方案 3 代价大，当前告警系统已有 SSE）

**您的决策**:
- [ ] 方案 1（初始加载，无实时更新）

#### Q4: 告警历史查询的保留期

**问题**：`TargetAlertRepo.ListHistory()` 默认查询全部历史数据，可能很大。

**三个方案**：

**方案 A：无限期保留**
```
- 所有告警永久存 DB
- 用户可查询任意时间范围
```
**缺点**：DB 体积会爆炸（一年可能 >10GB）

---

**方案 B：时间滑动窗口（推荐）**
```
- 只保留最近 90 天的告警
- 更早的告警定期清理
- 用户只能查询过去 7/30/90 天选项
```
**优点**：空间有限，性能稳定

---

**方案 C：分级保留**
```
- 最近 7 天：完整数据
- 8-30 天：汇总数据（按小时聚合）
- 31-90 天：汇总数据（按天聚合）
- 90+ 天：删除
```
**优点**：平衡存储和查询灵活性
**缺点**：实现复杂

---

**建议**：✅ 方案 B（时间滑动窗口 90 天）

**您的决策**:
- [ ] 方案 A（无限期保留）
- [ ] 方案 B（滑动 90 天）
- [ ] 方案 C（分级保留）
